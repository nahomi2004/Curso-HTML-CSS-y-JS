Tipo de listas

Existen dos tipos de listas: las ordenadas y desordenadas.
Las listas ordenadas, siguen un orden, son aquellas que tienen numeros o letras como viñetas.
Las listas desordenadas no siguen ningun orden en especifico, tiene puntos, vistos o cualquier simbolo como viñetas.

Formularios

Para entender como hacer formularios, pero sobre todo como funcionan, debemos tener en cuenta que cuando un usuario llena un formulario, estos datos son enviados a un servidor, donde seran verificados y le devuelve a la pagina una respuesta en base a los datos enviados. Para entenderlo mejor, tengamos en cuenta una pagina de inicio, comun y corriente:

Para crear formularios vas a necesitar de la siguiente estructura:


<form>
	<input type="#" id="#" name"#" value="#">
</form>

Que quiere decir type? Bueno en esta seccion se describe el tipo de formulario que deseamos, hay varios tipos, pero en una pagina de inicio suelen haber: text, password, submit.
Los formularios tipo texto son aquellos que reciben como entrada algo escrito por el usuario. Los tipo contraseña reciben una constraseña, obviamente. Los tipo submit que serian aquellos botones para enviar algo.
Bueno, aclarado eso, hay que aclarar que aquello que se envia a los servidores el el name y el value. Name, se refiere al nombre del valor, y value al valor en si.
Imaginemos que Stefano quiere ingresar a una pagina web cualquiera, y para ingresar le piden "Usuario" y "Contraseña". Lo que se le va a enviar al servidor sera lo siguiente: 


	name="Usuario" value="Stefano"
	name="Contraseña" value="12345"

Una vez entendido esto, vamos a codificar. Para hacer formularios hay varias etiquetas importantes e interesantes, estas son:

  • form: que es para dar a entender que se va a empezar con un nuevo formulario.
  • label: sirve para especificar lo que queremos que coneste el usuario.
  • input: se detalla el tipo de valores que se van a recibir, su id, y nombre.
  • fieldset: es una etiqueta que ayuda a separar el formulario en secciones.
  • legend: da un titulo a las secciones creadas.
Datos del usuario





Como nos podemos dar cuenta hay varias palabras reservadas utilizadas que no se han explicado, por ejemplo:

  • action: dentro se debe especificar el archivo del servidor al cual se le va a enviar las respuestas del formulario. Este archivo usualmente es un .php pero puede estar en otros lenguajes.
  • for: aqui se debe espefificar el id del input, para que el label pueda saber a que input esta haciendo referencia.

Tenemos mas tipos de input: date, file, radio. Y una nueva area de texto, que funciona para aquellas entradas de texto demasiado largas.

Luego pasamos con la input radio que nos ayuda a crear los famosos formularios de opcion multiple. Para crearlos solo necesitamos espeficificar el type y su id, y el label siempre debajo del input (por motivos esteticos), pero ojo, esto es para aquellas preguntas que permitan varias opciones como respuestas.
En el caso de ser preguntas con una sola respuesta se debe especificar el name, este debe ser igual en todas las input, pero no acaba ahi, en este caso se aumenta la palabra value, que va a servir para que el servidor sepa que opcion escogieron, en aquellas entradas donde el usuario ingresa el texto no es necesario lo de value porque el valor es aquel que escribe el usuario, sin embargo, en caso de las preguntas de opcion multiple el servidor no va a saber que escogio el usuario si es que no se le espeficica el value que contiene esta opcion.
En conclusion, en aquella seccion donde se permite que el usuario escojaa as opciones que desee faltaria value, para que el servidor sepa los valores.
